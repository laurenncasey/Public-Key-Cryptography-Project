Public Key Crytography

Have you ever entered your credit card information, excited to buy the new pony you just ordered on Ebay, and suddenly some hacker has all of your bank information and is stealing your money? No, it's not just bad luck, it's the way your payment was secured...or wasn't, in this case. The way things like payment, email, direct messages, and google searches are encrypted determines how easy it is for a hacker to intercept and alter. Public Key Cryptography is a big part of online security in todays world. In the content following, you will see how Public Key Crytography uses ideas in Abstract Algebra like modulus groups, coprimes, and gcd.

Descriptions

Encryption

Encryption is the process of converting data (plaintext) into another form, called cyphertext. Encrypted data can only be transformed back to the original data (or message) by authenticated users. For example...

plaintext + encryption = cyphertext
cyphertext + decryption = plaintext
"Hello" + Encryption Key = "2bhsghdki"
"2bhsghdki" + Decryption Key = "Hello"

Public Key Cryptography

Public Key Crytography uses a pair of keys, one public and one private for each pair. These keys are used to encrypt and decrypt messages sent between the holders of each pair. The public key holds the encryption algorithm, the private key holds the decryption algorithm. Each holder of the keys, owns the others decryption algorithm. Each crytography algorithm is generated based on mathematical problems. A sender can also use the private key along with the message being sent to send a digital signature. This is helpful for the receiver, so they can verify who send the message and which private key to use to decrypt the message. Some public key algorithms include key distribution and secrecy (Diffie-Hellman Key Exchange), some provide digital signatures (Digital Signature Algorithm), and some provide both (RSA).

Diffie-Hellman Key Exchange

DH Key Exchange is a cryptography algorithm that is only for secure key distribution. This allows for securely encrypted keys to be exchanged between communicating parites over a public channel (or network). The encrypted key exchange allows the conversation to not be decrypted by attackers. The communicating parties use public key cryptography to deliver messages. The image on the right provides a visual for the keys held by Alice (the sender), Bob (the reciever), and the public (potentially a hacker). Notice Alice has a private key, public key, and common key (as does Bob). Alice will use her private key (R) to encrypt a message, then send the message to Bob, who will use his private key (RGY) to decrypt Alices message. This way, the public can only see the common keys and public keys. Keep in mind this visual picture only represents the exchaning of keys and how they are generated - as does the Diffie-Hellman Key Exchange.

RSA

RSA is a cryptography algorithm that uses two linked keys to encrypt and decrypt messages being sent. This not only encrypts and decrypts data but can also digitally sign data as well. The key used to decrypt a message is opposite of the one to encrypt the message. In the image to the right, this demonstrates the simplistic process of RSA. Bob wants to send a message to Alice. He holds Alice's public key, so he encrypts the message with that. Though the image does not show, the key can also digitally sign the message so when Alice recieves it, she knows it came from Bob. Once Alice has received her message, she can use her private key to decrypt the message. With RSA, the key you use to encrypt is opposite from the key you use to decrypt. So, if Bob wanted, he could've encrypted the message using Alice's private key and sent the message. Then Alice would use her public key to decrypt the message. This (encryption by private key) is not commonly used because the public (anyone, including hackers) has Alice's public key. Meaning, if the messages encrypted by the private key was intercepted, anyone would be able to decrypt the message using the public key. By using the public key to encrypt instead, it allows for better security and confidentiality of the message being sent.

Digital Signatures

A digital signature is generated from the senders private key. When the sender sends the encrypted message, they send it along with their digital signature. Though, like the messsage is encrypted, so is the signature. The signature goes through a hash function. Then when the receiver gets the message, they can hash the signature using the same hash function to make sure it came from the right person. This process is known as a digital signature. Digital signatures authenticate users. Digital signatures cannot be modified by anyone, therefore forging signatures is not possible.

Hashing

A hashing function simple converts a given numeric or alphanumeric value into a small integer. There must exist a hash table that holds key-value pairs. The hashed message (now an integer) maps to the original message in the hash table. Using the hash function on a message will convert the message to an integer. Then using the hash function on the integer, converts the integer back to the message. This is by some algorithm (determined by the developer of the hash function) that uses the hash table as a mapping of values.

Padding Scheme

Padding Schemes are methods agreed upon by users that secure the message being sent. These schemes pad the message $m$ with some number of additional bits. The size of the unpadded message, $M$, must be somewhat smaller than $m$. Some examples of padding schemes are: PKCS5, One and Zeros, ANSI X9.23, W3C, etc.

Applications

Digital Signatures

A message can be digitally signed with the senders private key. The message is sent without any encryption (due to the digital signature process). After receiving the message, the receiver verifies the signature of the sender with their public key.

Digital Certificates

Since attackers can intercept messages and alter the message and/or key, digital certificates are used to authenticate the public key of a user. Before any public keys are on the network and in use, each user in the message transactions requests a certificate authority (CA) that issues a digital signature to their public key. This digital certificate may consist of information such as the users' public key, user name, user signature, etc.

Other

Today public key cryptosystems are used in online services such as emailing, online payment, websites (HTTPS), texts, etc.

Weakenesses

Public Key Cryptography

An attack can impersonate as an authenticated user. They can substitute the public key with a fake key. The can intercept messages and/or alter key. Online services of cryptosystems can only be secure when the public key is authentic and the user signatures are secure.

RSA

A big weakness of RSA is the fact that it's a slow algorithm (takes a while to excute). Due to the inefficiency of RSA, it's not ususally used to directly encrypt user data. Instead it's more used to transmit shared keys for symmetric key cryptography.

Algorithm

Diffie-Hellman (DH) Key Exchange

The original implementation of DH uses mutliplicative group of integers modulo p (where p is prime) and g (in group G) is a primitive root modulo p. In the following example of DH,

known values

(known by anyone) are in red and

unknown values

are in blue.

Alice and Bob want to communicate. Alice and Bob agree to

p = 23

and

g = 5

.
Alice chooses

a = 4

and sends Bob $A = g^{a}\;mod \;p $.
Bob chooses

b = 3

and sends Alice B = $g^{b}\;mod\;p$.
Alice computes $s=B^{a}\;mod\;p$ so $s=10^{4}\;mod\;23=18$.
Bob computes $s=A^{b}\;mod\;p$ so $s=4^{3}\;mod\;23=18$.

Alice and Bob have now shared a secret (the number 18). To decrypt the others' message, they solved:
$$A^{b}\;mod\;p\;=\;g^{ab}\;mod\;p\;=\;(g^{b}\;mod\;p)^{a}\;mod\;p\;=\;(g^{a}\;mod\;p)^{b}\;mod\;p$$

Keep in mind Alice and Bob would have to use larger values of $a$, $b$, and $p$ to make this encryption secure. In the case, since there are only 23 results of $n\;mod\;23$, this would not be very secure as there needs to be more than 23 results. If $p$ were a prime number of at least 600 digits, then even the fastest computer using the most optimal algorithm could not solve $a$ when only given $g$, $p$, and $g^{a}\;mod\;p$. For a hacker to intercept a message of DH Key Exchange they would have to solve the Diffie-Hellman Problem:
$$Given\;an\;element\;g\;and\;the\;values\;of\;g^{x}\;and\;g^{y},\;what\;is\;the\;value\;of\;g^{xy}?$$ Where g is the generator of the group and the group is the mutliplicative group of a finite field or elliptic curve.

In general, the description of the protocol (DH) is as follows:

Alice and Bob agree on a finite cylic group $G$ under multiplcation, where $|G|=n$ and

$\lt g \gt \in G$.


Alice picks

$a \in \mathbb{N} \; such\;that\; 1 \lt a \lt n$

and sends $g^{a} \in G$ to Bob.
Bob picks

$b \in \mathbb{N} \;such\;that\; 1 \lt b \lt n$

and sends $g^{b} \in G$ to Alice.
Alice computes $ (g^{b})^{a}\;=\;g^{ba} \in G$
Bob computes $(g^{a})^{b}\;=\;g^{ab} \in G$
Now Alice and Bob hold the group element $g^{ab} = g^{ba}$, which now serves as the shared secret key to decrypt messages.

RSA

A RSA user creates and publishes a public key based on two large prime number. The prime numbers are kept secret. Messages may be encrypted by anybody, with the public key, but only decrypted by those who know the chosen prime numbers. The security of this algorithm relies on the "factoring problem", that is the difficulty of factoring the product of two large prime numbers. To break this security, one would have to solve the RSA Problem (shown below). There are no current published methods to defeat the system, as long as a large enough key is used.

RSA Problem

There are four steps to the RSA Problem: key generation, key distribution, encryption, and decryption. The basic principal is to find three very large positive integers $e$, $d$, and $n$ such that with modular exponentiation for all integers $m$ ($0 \leq m \leq n$): $$(m^{e})^{d} \equiv m \;(mod\; n)$$ Even knowing $e$, $n$ or even $m$, it can be very difficult to find d. The public key is represented by integers $n$ and $e$ and the private key by $d$ and $n$. $m$ represents the message. The four step process is as follows:

  1. Choose two large prime numbers $p$ and $q$.
    1. $p$ and $q$ should be chosen at random, be similar in magnitude but differ in length.
    2. $p$ and $q$ should be kept a secret
  2. Compute $n=pq$.
    1. $n$ is used as the modulus for both public and private keys. It's length (in bits) is the key length.
    2. $n$ is released to the network as part of the public key.
  3. Compute $\alpha (n)$ where $\alpha$ is Carmichaels Totient Function.
    1. Since $n=pq$, $\alpha (n) = lmc(\alpha (p), \alpha (q))$ and since $p$ and $q$ are prime, then $\alpha (p) = \theta (p) = p-1$ and $\alpha (q) = \theta (q) = q-1$. Hence $\alpha (n) = lmc(p-1, q-1)$.
    2. $lmc$ can be calculated through the Euclidean algorithm $lmc(a,b)= \frac{|ab|}{gcd(a,b)}$.
    3. $\alpha (n)$ is kept secret.
  4. Choose integer $e$ such that $1 \lt e \lt \alpha (n)$ and $gcd(e, \alpha (n) ) = 1$ (that is, $e$ and $\alpha (n)$ are coprime).
    1. $e$ should have a short bit length and small Hamming weight. This results in efficient encryption. Most commonly chosen value for $e$ is $2^{16} + 1$. The smallest and fasted value for $e$ is 3, but with such a small value it is less secure.
    2. $e$ is released on the network as part of the public key.
  5. Determine $d$ as $d \equiv e^{-1} \;(mod\; \alpha (n))$, that is $d$ is the mutliplicative inverse of $e$ modulo $\alpha (n)$.
    1. Solve for $d$ in the equation $de \; mod \; \alpha(n) = 1$. Since $e$ and $\alpha (n)$ are coprime, then the equation is a form of Bezouts identity, where $d$ is one of the coefficients.
    2. $d$ is kept secret as the private key exponent.

Example

Key Distribution

Alice and Bob want to communicate. Bob needs Alices' public key so he can send a message to Alice. Alice transmits her public key $(n,e)$ to Bob via a reliable (but not necessarily secret) route. Alices' private key $d$ is never distributed.

Encryption

For Bob to encrypt the message $M$, he turns $M$ into an integer $m$ such that $0 \leq m \leq n$ by using an agreed upon reversible protocol known as a padding scheme.He then computes the ciphertext $c$ using Alices' public key $e$: $$c \equiv m^{e} \; mod\; n$$ Bob then transmits the message to Alice.

Decryption

Alice can recover $m$ from $c$ by using her private key exponent $d$ by computing: $$c^{d} \equiv (m^{e})^{d} \equiv m \;(mod \; n)$$ Given $m$, she can recover the original message $M$ by reversing the padding scheme.

History

Public Key Cryptography

Initally, symmetric crytography was used. This is where one key (algorithm) is sued for both the encryption and decryption of the message (or data). If the key is kept secure, the the data is secure. The biggest issue with the symmetric cryptosystem is the sharing of the key. It became difficult to securely share the key from sender to reciver. So the invention of the public key cryptosystem was made. Public key cryptography is considered the most secure cryptography to create digital signatures and perform encryption. The idea of public key cryptography is attributed to Whitfield Diffie and Martin Hellman who published the concept in 1976. They also have an algorithm (Diffie-Hellman Key Exchange) they published as well, as one of the first public key protocols.

RSA

RSA comes from the surnames Ron Rivest, Audi Shamir, and Leonard Adleman who publically described the cryptography algorithm in 1977. The same system was secretly developed in 1973 by a mathematician, Clifford Cocks, who worked for a British intelligence agency. Cliffords' sysytem was declassified in 1977.

Proof of Question 1

Proof
Assume $p = 5$, $q = 7$. Using RSA, we want to determine whether $e = 10$ or $e = 11$ will work. By the process of RSA, $n = pq = 35$ and $\alpha(n) = lcm(p-1, q-1) = lcm(4,6) = 12$. Then we want to pick an $e$ such that $e$ and $\alpha(n)$ are coprime. That is, by definition of coprime, $gcd(e, \alpha(n)) = 1$. Notice $gcd(10, 12) = 2$ and $gcd(11, 12) = 1$. This means $e$ must equal 11. Now we want to show what the private and public keys are. By RSA, the public key is (e, n) and the private key is (d, n). We must choose d such that $de \; mod \; \alpha(n) = 1$. Let $d = p[pe \; mod \; \alpha(n)] = 5[55 \; mod \; 12] = 5[7] = 35$. Then the public key is $(e,n) = (11, 35)$ and the private key is $(d, n) = (35, 35)$.
Q.E.D.



Proof of Question 2

Proof
Let m = 3. We want to show the ciphertext (c) and plaintext (p). By RSA, $c = m^{e} \; mod \; n$ (by using the public key). So $c = 3^{11} \; mod \; 35 = 12$. Therefore the sent encrypted message is the number 12. Then using the private key to decrypt this, we get $m = c^{d} \; mod \; n = 12^{35} \; mod \; 35 = 3$. Therefore the decrypted message, 3, equals the sent message, 3.
Q.E.D.



Proof of Question 3

Proof
Let $p = 7$ , q = $13$, $e = 5$, and $m = [8, 71, 71, 44, 1, 51]$. We want to show the decoded ciphertext. By RSA, $n = pq = 91$ and $\alpha(n) = lcm(p-1, q-1) = lcm(6, 12) = 12$. Next we must compute the private key (d, n) such that $de \; mod \; \alpha(n) = 1$. Let $d = p[pe \; mod \; \alpha(n)] = 7[7*5 \; mod \; 12] = 7[11] = 77$. Notice $de \; mod \; \alpha(n) = 77*5 \; mod \; 12 = 385 \; mod \; 12 = 1$. So our public key is $(e,n) = (5, 91)$ and our private key is $(d, n) = (77, 91)$. To decrypt the encrypted message m, and therefore show the decoded ciphertext, we must take each integer in m and apply the private key to it. That is, for each integer in m, we are computing $c = m_{1}^{d} \; mod \; n$ where m_{1} is an integer in m. By doing this, we get c = [8, 15, 15, 18, 1, 25]. Then by using our determined algorithm above, our received message is "hooray".
Q.E.D.



Proof of Question 4

Proof
Let p = 13, g = 7, and a = 5. We want to use the received message, $B = g^{3} \; mod \; 13$ to determine the shared key. By Diffie Hellman Key Exchange, we can compute $B = 7^{3} \; mod \; 13 = 5$. Then we can compute $s = B^{a} \; mod \; p= 5^{5} \; mod \; 13= 3125 \; mod \; 13 = 5$. Thus the shared key between you and your friend was the number 5.
